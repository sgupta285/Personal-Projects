# E-Commerce Conversion Optimization

A product analytics and experimentation project focused on where users leak out of an e-commerce funnel and whether a treatment meaningfully improves conversion.

The repository starts with synthetic event-level session data and turns it into warehouse-style SQL metrics, experiment cohorts, statistical readouts, segment-level lift tables, business recommendations, and a dashboard. The goal is to make the repo feel like a serious growth analytics project instead of a toy notebook.

## Funnel

`landing -> product_page -> cart -> checkout -> purchase`

Each session is assigned to either `control` or `treatment`. The treatment represents a revised product page plus a simpler cart handoff. The synthetic effect is modest overall and stronger in higher-friction segments, which makes the segment analysis more useful than a single top-line number.

## What the repo includes

- synthetic e-commerce sessions and event stream
- SQL models for funnel stage metrics and experiment cohorts
- reusable A/B testing utility module
- segment-level lift analysis by device, channel, recency, and category
- business recommendation memo and findings write-up
- Streamlit dashboard for leakage and lift monitoring
- generated metrics, plots, and benchmark artifacts
- tests for core conversion logic

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
cp .env.example .env
make bootstrap
```

Run the dashboard:

```bash
make dashboard
```

Run the tests:

```bash
make test
```

## Notes on methodology

- SQL is used as the source of truth for the core funnel and cohort tables.
- The primary experiment KPI is purchase conversion rate.
- The report also tracks order value, revenue per session, confidence intervals, observed power, and MDE.
- Segment-level lift is included because aggregate results can hide where the treatment actually works.
- The data is synthetic, but the measurement workflow mirrors a real growth or product analytics project.
